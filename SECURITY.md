# Security Policy

## 🔒 Supported Versions

The following table lists the versions of KiteML currently receiving security updates:

| Version | Supported | Security Maintenance |
| :--- | :---: | :--- |
| `1.0.x` | ✅ Yes | Active security patches and vulnerability resolution |
| `< 1.0.0` | ❌ No | Deprecated prototype releases |

---

## 🛡️ Reporting a Vulnerability

The KiteML team takes the security of our open-source framework and users seriously. If you discover a security vulnerability in KiteML (such as code execution in serialization, REST serving vulnerabilities, or dependency CVEs), **please do not create a public GitHub issue**.

Instead, please report security vulnerabilities directly to the core maintainer team at:

📧 **`priyathamprime7@gmail.com`**

### What to Include in Your Report
- A detailed description of the vulnerability and potential security impact.
- Step-by-step instructions or a minimal reproducible python script.
- Affected KiteML version(s), Python version, and operating system.

---

## ⏱️ Response & Disclosure Policy

- **Initial Acknowledgement**: We will acknowledge receipt of your vulnerability report within **48 hours**.
- **Assessment & Patch**: We will investigate and provide a timeline for a security patch within **7 days**.
- **Public Release & Disclosure**: Once a patch is released, we will credit your contribution in the security release notes.
