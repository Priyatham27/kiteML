# Security Policy

## 🔒 Reporting Security Vulnerabilities

The KiteML maintainers take the security of our framework and users seriously. If you discover a security vulnerability in KiteML, please **do not create a public GitHub issue**.

Instead, please report security vulnerabilities responsibly by emailing the core security team at:

**`priyathamprime7@gmail.com`**

### What to Include in Your Report
- Description of the vulnerability and potential impact.
- Steps to reproduce or a proof-of-concept script.
- Affected KiteML versions or components (e.g. `serving`, `serialization`).

### Response Expectations
- We will acknowledge receipt of your vulnerability report within **48 hours**.
- We will provide a status update and estimated patch release date within **7 days**.
- Credit will be given in release notes upon publication of the patch.
