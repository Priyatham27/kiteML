# KiteML Logo Usage & Brand Guidelines (`brand/logo-usage.md`)

This guide establishes clear rules for using the KiteML logo, icon, and visual assets across digital and print media.

---

## 1. Clearspace Requirements

To maintain visual integrity and impact, keep a minimum clearspace around the logo equal to **`1X`** (where `X` equals the height of the icon):

```text
    ┌───────────────────────────┐
    │          1X Space         │
    │  1X ┌───────────────┐ 1X  │
    │     │ 🪁 KiteML     │     │
    │  1X └───────────────┘ 1X  │
    │          1X Space         │
    └───────────────────────────┘
```

---

## 2. Minimum Render Sizes

| Asset | Minimum Width | Context |
| :--- | :--- | :--- |
| **Favicon Icon (`favicon.svg`)** | `16 × 16 px` | Browser tabs & taskbar icons |
| **Standalone Icon (`kiteml-icon.svg`)** | `32 × 32 px` | Mobile app header, CLI icons |
| **Full Primary Logo (`kiteml-logo.svg`)** | `120 × 32 px` | Documentation header, navbar |

---

## 3. Brand Rules: Do's and Don'ts

### ✅ DO:
- Use the **Dark Logo** (`kiteml-logo-dark.svg`) on dark backgrounds (`#0B0F19`).
- Use the **Light Logo** (`kiteml-logo-light.svg`) on light backgrounds (`#FFFFFF`).
- Maintain proportional aspect ratios when scaling vector files.

### ❌ DON'T:
- **Do NOT** add drop shadows, outer glows, or 3D bevel effects to the icon.
- **Do NOT** introduce color gradients to the diamond polygons or circuit lines.
- **Do NOT** stretch, distort, or rotate the diamond kite geometry.
- **Do NOT** change the `KiteML` wordmark colors (e.g. turning "ML" red or yellow).
- **Do NOT** overcomplicate or modify the circuit node structure.
